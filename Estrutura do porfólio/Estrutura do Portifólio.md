# Estrutura do Repositório no GitHub

Cada aluno (ou grupo) terá um repositório com estrutura base como:

```
conectajovem-portfolio/
│
├── README.md    → apresentação do aluno
├── artigos/
│   ├── seguranca-online.md
│   ├── hardware-basico.md
│   ├── sistema-operacional.md
│   └── cidadania-digital.md
│
├── projetos/
│   ├── planilha-orcamento.md
│   ├── apresentacao-powerpoint.md
│   └── site-pessoal-markdown.md
│
└── img/
    ├── seguranca-online.png
    ├── captura-terminal.png
    └── diagrama-hardware.jpg
```

## Por que usar Markdown (.md)

- É leve, simples e universal — usado em documentação profissional (GitHub, ReadTheDocs, Notion, etc.).

- Ensina organização, clareza e formatação técnica.

- Facilita a publicação no GitHub (páginas automáticas, histórico de commits, versionamento).

- Permite combinar texto, imagens e código em um único formato.

- Escrita técnica e documentação — habilidades valiosas para o mercado.

- Colaboração em ambientes de versionamento (Git).

- Reflexão crítica sobre tecnologia e sociedade.

- Autonomia digital e protagonismo.
